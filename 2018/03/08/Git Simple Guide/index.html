Git Simple Guide

Panduan buat belajar cara pake git
Baca Hayati Cobain :D

> Download Bahan

Sebelum pake git kita harus download dan install dulu pastinya, link nya dibawah ini

Buat Window
Buat OSX
Buat Linux

>Bikin Repository Baru

Awal awal kita tentu harus buat repository ,itu tempat buat menyimpan project kita ,command nya
git init

>Duplikasi Repository

Kita juga bisa menduplikasi repository nya,tujuannya misal untuk membuat cadangan apabila ada kerusakan atau hal lain
git clone /path/to/repository

kalau kamu menggunakan remote server
git clone username@host:/path/to/repository

>Workflow/Alur Kerja

install git udah,bikin repo udah,duplikasi udah,sekarang kita harus tau dulu workflownya,lokal repository kita terdiri dari 3 bagian secara umum, direktori kerja untuk menyimpan file projek kita , Indeks untuk untuk pengolah data dab Head yang menujukan commit terakhir kita.

>Add & Commit

Add dan Commit ,kedua hal itu digunakan menyimpan sebuah “ checkpoint “ buat projek kita,Add untuk memulai workflow git karena add digunakan untuk penambahan ke Indeks command nya
git add <namaberkas>
git add *

kemudian commit untuk menyimpan dan memberi deskripsi dari “ checkpoint “, commandnya
git commit -m "Deskripsi"

dengan kedua data tersebut data telah tersimpan di “ checkpoint “ lokal.

>Pushing Changes/Kirim Perubahan

Sebelumnya kita sudah bisa menyimpan hasil commit di lokal,kita juga bisa mengirimnya ke repository di remote server commandnya
git push origin "nama branch"

Tapi bagaimana jika repository belum di duplikasi dan tetap ingin dihubungkan dengan dengan remote server?kita bisa menggunakan command dibawah
git remote add origin <nama server>

>Pulling Changes/Mengambil Perubahan

Jika ada mengirim tentu ada mengambil,di git juga sama,caranya
git pull

Dan semua perubahan di remote server akan tersimpan di lokal

>Branch

Diatas telah disebutkan branch ,namun diberi penjelasannya,sebenarnya branch seperti namanya adalah percabangan,ketika kita membuat projek kita terkadang ingin menambahkan fitur atau apapun namun tidak ingin mengganggu dari fungsi yang sudah ada,
tapi kemudian kita ingin menggunakan fitur tsb di projek kita,kita bisa menggabungkan branch master atau branch utama ke branch tsb,
pertama kita masuk ke branch yang ingin di merge
git checkout "nama branch"

kemudian kita merge atau gabung dengan branch utama misal, command nya
git merge "branch"

Sebelumnya kita juga bisa mengecek untuk mengantisipasi error saat merging,caranya
git diff "branch asal" "branch tujuan"

>Tagging

Ketika kita commit kita akan mendapatkan id dari dari checkpoint kita, kita bisa menandai nya untuk mempermudah pemanggilan atau proses yang menggunakan ID kedepannya,caranya
git tag "nama tag" "ID commit"

>Log

Kita juga bisa melihat history dari repository kita, temasuk semua commit yang kita buat,secara umum caranya
git log

kita ingin menampilkan secara spesifik,penulis tertentu misal,bisa tentu saja,caranya
git log --author="nama"

atau ingin log ditampilkan hana satu line, bisa, caranya
git log --pretty=oneline

tambah dengan pohon ascii dan nama dan tagnya juga,bisa,caranya
git log --graph --oneline --decorate --all

masih banyak sebenarnya tapi bisa di cek sendiri ,caranya buka
git log --help

>Reset

Adakalanya kita ingin mengembalikan projek kita ke kondisi terakhir sebelumnya,kita bisa menggunakan
git checkout -- <namaberkas>

tapi bisa jadi kita ingin menghapus semua perubahan di lokal dan ingin menggunakan projek di remote server seluruhnya,caranya dengan mengambil semua commit di server dan kemudian menimpa semua yang ada di lokal,commandnya
git fetch origin
git reset --hard origin/master
